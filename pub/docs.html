Getting Started

In this page you will see a short overview to get started with PermForm followed by a detailed API documentation.

Installation

To use PermForm, you must first include the JavaScript file in the <head> section of your web page: 
<script defer type="text/javascript" src="PermForm.js"></script>

Sample code

A code snippet of some code that creates a form inside the div with ID "simple-form" which must already exist on your page. 
initFormSimple("simple-form", ["textLabel","textInput","fileInput","dropDown","submit",]).buildForm();

Functions

All the functions available once you include the PermForm library.

Name Arugments Returned Behavior
initFormSimple divId, elements PermForm object Creates a PermForm object. Elements must be an array of element names. Look at the elements array API table for more details.
initForm specs PermForm object Creates a PermForm object. Specs must be a Specs object with specification options and an element array. Look at the specs API table for more details.

Methods

All the methods you can call on a PermForm object.

Name Arugments Returned Behavior
buildForm Builds the form inside the div specified by divId in the specs given to initialize the PermForm object. Uses properties in the specs object to style the form. Looks through the elements array which must contain element objects (see the elements array API table for more details).
rebuildForm specs Updates the specs object of the PermForm object, removes the old form build using buildForm (if any) and rebuilds a new form with the new specs. Note, the divId in this specs is disregarded, and the old div provided during initialization of the PermForm object will be used.
exportForm Specs object Returns the specs object of the form. Used in conjunction with the draggability function to style the form, and export it's JSON for later use.
exportState array Returns an array of the current state of each element on the form obtained via user input.

Specs Object

All the key-value pairs of a specs object. Any entry with a default value of '-' indicates that the field has no default and is a required key.

Key Default Type Purpose
divId - String The ID of the div element the form must be built into. A div must already exist with this ID, and must be positioned and sized as wanted.
autoSave false Boolean Whether or not to automatically save the state of each form element persistently. With this enabled, whenever the user visits the site again, the form elements are automatically filled in with the saved information.
draggable false Boolean Whether or not the form elements can be dragged around.
theme basic String The theme of the form. Currently available themes: basic.
columns basic Integer The number of columns of the form.
elements - Array An array of element objects to be a part of the form. See the elements array API table for more details.
onSubmit - Callback A callback called when a user clicks the form element type 'submit' on the form. An array of the current state of each form element is passed as an argument.

Elements Array

A 2D array of element specification objects. Each index of this array is an array and specifies a row in the form and each index in that row array is a form element on the form. Each object in the element array is an element specification object and details about this object is given below. The common properties are listed below.
The 'type' property corresponds to the 'Type' header in the table below.
The 'size' property of each object is a comma separated coordinate form, with the horizontal and vertical sizes of the element.
The 'required' property is a Boolean, and indicates whether or not a field is required.

Type Keys Default Behavior
textLabel type, value, size type: "textLabel"
value: "Text Label"
size: "1,1" 		A simple text label. Displays the text given by the value property.
textInput type, name, required, validator, size type: "textInput"
placeholder: "Text Input"
size: "1,1" 		A simple text input box. Placeholder text is given by placeholder. Provide a vertical sizing of > 1 for multiline input. Validator indicates how to validate this input. Currently supported validators: 'email', 'numeric'.
fileInput type, name, required, size type: "fileInput"
name: "File Input"
size: "1,1" 		A file input object. Users must click on this object to select and upload a file from the file selector window. Text displayed on the file input object is given by the property name.
dropDown type, name, options, size type: "dropDown"
name: "Drop Down"
options: ["Yes", "No"]
size: "1,1" 		A drop down selection object. Text displayed on the object is given by the property name. The propery options is an array of strings corresponding to the available options for the user.
submit type, size type: "submit"
size: "1,1" 		A submission button. This calls the 'onSubmit' property of the form specifications with an array of the states of each element on the form.
